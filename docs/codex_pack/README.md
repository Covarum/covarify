# Covarify Master Playbook v1.0 - Codex Source Pack

This folder is the canonical product and company source for Covarify.

## Primary instruction
Read `COVARIFY_MASTER_PLAYBOOK_v1.0.md` before making material product, UX, financial-decision, trust, data, AI, voice, memory, or business changes.

## Locked brand source
`COVARIFY_FINAL_BRAND_BOARD.jpeg` is the finalized visual identity. Do not redraw, simplify, substitute, or reinterpret the logo without founder approval.

## Governing rules
- Financial facts, scenarios, decisions, recommendations, memory observations, and outcomes are distinct records.
- Conversation is the interface, but confirmation is the control.
- Plaid is an input, not the product.
- Rules and deterministic financial logic come before AI explanation.
- No silent state changes.
- No shame, hidden assumptions, or false precision.
- Protect essentials and required minimums before optimization.
- Every recommendation must expose evidence, assumptions, missing information, tradeoffs, confidence, and review timing.
- The user remains in control and meaningful changes must be traceable and reversible where possible.

## Change protocol
When a material product decision changes:
1. Update the playbook source.
2. Add or update the Decision Register.
3. Add any new failure mode to the Risk Register.
4. Record the implementation commit and review date.
5. Never silently replace a locked decision.
